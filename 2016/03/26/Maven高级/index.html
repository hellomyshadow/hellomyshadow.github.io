Maven高级

发表于 2016-03-26 | 分类于 JavaEE

SSM整合

1. 引入SSM、log4j、shiro、spring-security的相关依赖
    1. aspectjweaver、spring-aop、spring-context、spring-context-support、spring-web
    2. spring-orm、spring-beans、spring-core、spring-test、spring-webmvc、spring-tx
    3. junit(test)、、mysql-connector-jave、servlet-api(provided)、jsp-api(provided)、jstl
    4. log4j、slf4j-api、slf4j-log4j12、mybatis、mybatis-spring、c3p0(type:jar，scope:compile)、
    5. pagehelper、spring-security-web、spring-security-config、spring-security-core、
    6. spring-security-taglibs、druid
2. 使用Maven的web骨架，创建JavaWeb工程;
3. 把 log4j.properties 放在 resources 目录中
    # Set root category priority to INFO and its only appender to CONSOLE.
    #log4j.rootCategory=INFO, CONSOLE            debug   info   warn error fatal
    log4j.rootCategory=debug, CONSOLE, LOGFILE
    # Set the enterprise logger category to FATAL and its only appender to CONSOLE.
    log4j.logger.org.apache.axis.enterprise=FATAL, CONSOLE
    # CONSOLE is set to be a ConsoleAppender using a PatternLayout.
    log4j.appender.CONSOLE=org.apache.log4j.ConsoleAppender
    log4j.appender.CONSOLE.layout=org.apache.log4j.PatternLayout
    log4j.appender.CONSOLE.layout.ConversionPattern=%d{ISO8601} %-6r [%15.15t] %-5p %30.30c %x - %m\n
    # LOGFILE is set to be a File appender using a PatternLayout.
    log4j.appender.LOGFILE=org.apache.log4j.FileAppender
    log4j.appender.LOGFILE.File=d:\axis.log
    log4j.appender.LOGFILE.Append=true
    log4j.appender.LOGFILE.layout=org.apache.log4j.PatternLayout
    log4j.appender.LOGFILE.layout.ConversionPattern=%d{ISO8601} %-6r [%15.15t] %-5p %30.30c %x - %m\n
4. Spring的配置文件：applicationContext.xml
5. Spring整合Mybatis时，借助 mybatis-spring 包;
    1. SqlSessionFactoryBean：SqlSessionFactory的工厂对象;
    2. MapperScannerConfigurer：扫描Dao接口的包，为所有的Dao生成代理对象;
    3. 使用Dao映射配置文件的方式时，映射文件的包路径与Dao的包路径一致，文件名与映射的Dao接口名一致;
    4. 比如cn.test.dao.IUserDao，对应的映射配置文件：resources/cn/test/dao/IUserDao.xml;
    5. Druid是阿里提供的连接池，其<property>的配置和c3p0一样.
6. Service层使用注解开发，在 applicationContext.xml 中配置只扫描Service包，配置事务管理器、事务通知、切面;
    <context:component-scan base-package="cn.test.service" />
    @Service
    public class UserServiceImpl implements IUserService {
        @Autowired
        private IUserDao dao;
        ......
    }
7. SpringMVC的配置文件：resources/springmvc.xml
    1. ，配置前端处理器、视图解析器、扫描Controller层所在包、并开启注解扫描的支持
    <context:component-scan base-package="cn.test.controller" />
    <!-- 简化的配置方式：配置处理器映射器和处理器适配器！不需要配置前端处理器？？？ -->
    <mvc:annotation-driven />
    <!-- 视图解析器 -->
    <bean id="internalResourceViewResolver" 
            class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/pages" />
        <property name="suffix" value=".jsp" />
    </bean>
    <!-- 过滤掉静态资源，交给默认的Servlet去处理，但是他怎么知道哪些是静态资源，竟然不需要配置<mvc:resources> -->
    <mvc:default-servlet-handler />
8. web.xml
    1. 配置编码过滤器CharacterEncodingFilter，避免中文乱码;
    2. 配置Spring的核心监听器，并重新指定Spring配置文件的路径，加载Spring的配置文件;
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>
    <context-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:applicationContext.xml</param-value>
    </context-param>
    3. 配置SpringMVC的核心Servlet，在服务器启动时，创建此Servlet对象，并加载SpringMVC的配置文件;
    <servlet>
        <servlet-name>dispatcherServlet</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:springmvc.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>dispatcherServlet</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
9. 配置Tomcat7插件，使用 tomcat7:run 运行，http://localhost:8080/项目名/ 访问;

拆分与聚合

1. Maven工程的拆分：把一个完整的Maven项目分成不同的独立模块，这些模块以坐标的形式依赖;
2. 聚合：把拆分成零散的模块，聚合成一个完整的项目工程;
3. 拆分：
    1. 创建父工程：不使用Maven骨架，只需保留一个pom.xml，其他的文件和目录都可以删除掉;
    2. 创建子工程(项目模块)：右机父工程 --> New Module --> 创建Maven项目;
    3. Dao模块和Service都不使用骨架，Controller使用web骨架;
    4. 子工程的 pom.xml 中，会有一个 <parent> 节点，表示继承的父工程;
    5. 父工程的 pom.xml 中，会多出一个 <modules> 节点，列出拥有的子模块;
4. 模块的关系
    1. 模块之间是平级的，需要建立依赖关系，才能引用对方的功能;
    2. 比如，Service模块引用Dao模块的坐标，Controller模块引用Service模块的坐标;
    3. 如果想要引用另一个工程，则把工程打包发布到本地仓库中，再引用工程的坐标即可建立依赖关系;
5. 父子工程具有继承关系，子工程通过传递依赖，能够直接使用父工程的依赖包;
    1. 把公共的jar包依赖放在父工程中，供所有的子工程使用;
    2. 默认情况下，依赖的作用域(scope)都是compile，如果传递依赖的作用域是test，传递过程中可能丢失;
    3. 比如父工程引入了junit(test)，子工程可能无法使用，这种情况下，通常会在子工程中再加入junit依赖即可;
6. 配置文件的拆分：拆分Spring的配置文件applicationContext.xml
    1. Dao模块部分：resources/spring/applicationContext-dao.xml：添加命名空间和约束，只配置Dao相关;
    2. Service模块部分：resources/spring/applicationContext-service.xml：只配置Service相关;
    3. Controller模块：resources/applicationContext.xml，添加命名空间和约束，使用 <import> 导入
        <import resource="classpath:spring/applicationContext-dao.xml" />
        <import resource="classpath:spring/applicationContext-service.xml" />
    4. Controller工程依赖于Service工程，Service工程依赖于Dao工程，最终Spring的配置文件会编译成一份;
7. Controller模块中，把 log4j.properties 和 springmvc.xml 放在 resources 目录中;

启动项目

1. 方式一：打开右侧的Maven面板 --> 展开父工程的Plugins --> 双击 tomcat7:run 启动;
2. 方式二：配置本地的Tomcat，启动工程;
3. 方式三
    1. 如果Controller工程的tomcat7:run，启动会报错，是因为Maven在启动项目时，会扫描项目的pom.xml时，
    查看本地仓库是否存在引入的依赖，如果没有，则从中央仓库下载，如果找不到，则报错;
    2. Controller工程中引入了Service工程的坐标，但Service工程是本地项目，没有发布到本地仓库，所以启动报错;
    3. 由此可知，只需要把Service工程打包发布到本地仓库即可，但是Service工程又引入了Dao工程的坐标，
    所以只能去打包父工程;
    4. 在右侧的Maven面板上，双击父工程的Lifecycle --> install，安装成功后，启动Controller工程的tomcat7:run;

私服

搭建私服(windows)

1. 使用专门的Maven仓库管理软件来搭建私服，比如：Apache Archiva，Artifactory，Sonatype Nexus
2. 选择Sonatype Nexus，下载开源版的Nexus压缩包，解压后有两个目录：应用程序目录，私服仓库目录(sonatype-work);
3. 以管理员模式启动cmd窗口，进入应用程序目录，执行bin目录下的 nexus.bat 命令;
    1. 安装、卸载、运行：nexus.bat install，nexus.bat uninstall，nexus.bat start
    2. 查看启动后的服务：右击计算机 -> 管理 -> 服务和应用程序 -> 服务，在列表中查找 nexus;
4. nuxus的全局配置文件：conf/nexus.properties，默认使用的端口号是8081;
5. 启动图形化界面时，确保8081端口未被占用：http://localhost:8081/nexus
    1. 默认用户名和密码：admin，admin123
    2. 左侧导航栏：点击Repositories --> 浏览仓库，查看仓库信息、地址;
    3. Add添加仓库，选择仓库类型：Hosted、Proxy、Virtual，还可以创建仓库组;
6. 创建不同的仓库，管理不同的依赖包
    1. 存放测试包的仓库Snapshots(hosted)，发布包的仓库Releases(hosted);
    2. 第三方包的仓库3rd party(hosted)，中央仓库的代理仓库Central(proxy);
    3. 创建仓库组，把已有的仓库按顺序添加到组中，在本地仓库在私服中查找依赖包时，就能按照仓库的顺序查找，
    最后由代理仓库转到中央仓库中去查找;

发布私服

1. Maven的 settings.xml，配置私服仓库的信息
    <servers>
        <server>
            <id>releases</id>  ===>指定私服仓库的ID
            <username>admin</username>  ===>私服的用户名密码
            <password>admin123</password>
        </server>
        <server>
            <id>snapshots</id>  ===>指定私服仓库的ID
            <username>admin</username>
            <password>admin123</password>
        </server>
    </servers>
2. 要发布哪个项目/模块，就配置它的 pom.xml
    <distributionManagement>
        <!-- 配置发布的仓库ID和地址 -->
        <repository>  ===>正式版的配置，明确发布到存放正式包的仓库
            <id>releases</id>  ===>此ID必须在 settings.xml 中已经配置过了
            <url>http://localhost:8081/nexus/content/repositories/releases/</url>
        </repository>
        <snapshotRepository>
            <id>snapshots</id>  ===>测试版的配置，明确发布到存放测试包的仓库
            <url>http://localhost:8081/nexus/content/repositories/snapshots/</url>
        </snapshotRepository>
    </distributionManagement>
    1. 执行Lifecycle --> deploy
    2. 执行前面的生命周期时，其后面的生命周期也会执行，所以 install 命令的执行，会把项目/模块安装到本地仓库;

从私服下载jar

Maven的 settings.xml
    <profiles>
        <profile> 
            <id>dev</id>  ===>profile的id
            <repositories>
                <repository>  ==>仓库id，repositories可以配置多个仓库，保证id不重复
                    <id>nexus</id>  ==>仓库地址，即nexus仓库组的地址
                    <url>http://localhost:8081/nexus/content/groups/public/</url>
                    <releases>
                        <enabled>true</enabled> ==>是否下载releases构件
                    </releases>
                    <snapshots>
                        <enabled>true</enabled>  ==>是否下载snapshots构件
                    </snapshots>
                </repository>
            </repositories>
            <pluginRepositories>  ==>插件仓库，Maven的运行依赖插件，也需要从私服下载插件
                <pluginRepository> ==>插件仓库的id不允许重复，否则后者会覆盖前者
                    <id>public</id>  ==>配置为仓库组，仓库组的id
                    <name>Public Repositories</name>  ==>仓库组的名字
                    <url>http://localhost:8081/nexus/content/groups/public/</url>  ==>仓库组的地址
                </pluginRepository>
            </pluginRepositories>
        </profile>
    </profiles>
    <activeProfiles>
        <activeProfile>dev</activeProfile>  ==>激活id=dev的profile
    </activeProfiles>

按照第三方jar包

1. 把第三方jar包安装到本地仓库，如fastjson.jar
    1. 查看fastjson.jar的坐标：groupId、artifactId、version;
    2. 进入fastjson.jar所在的目录，执行命令:
    mvn install:install-file -DgroupId=com.alibaba -DartifactId=fastjson -Dversion=1.1.37 
    -Dfile=fastjson-1.1.37.jar -Dpackaging=jar
    3. 如果不在fastjson.jar的目录，则需要指定绝对路径： -Dfile=D:\download\fastjson-1.1.37.jar
2. 把第三方jar包安装到私服
    1. 在Maven的 settings.xml 中，添加存放第三方jar包的仓库信息
        <server>
            <id>thirdparty</id>
            <username>admin</username>
            <password>admin123</password>
        </server>
    2. 进入jar所在目录
        mvn deploy:deploy-file -DgroupId=com.alibaba -DartifactId=fastjson -Dversion=1.1.37 
        -Dfile=fastjson-1.1.37.jar -Dpackaging=jar  -DrepositoryId=thirdparty 
        -Durl=http://localhost:8081/nexus/content/repositories/thirdparty/
        1. -Durl：私服的仓库地址;
        2. -DrepositoryId：私服的仓库ID;
    3. 如果不在jar包所在目录，-Dfile=jar包的绝对路径;
