SSM整合

1. 表现层SpringMVC，业务层Spring，持久层MyBatis
2. 整合方向：用Spring框架去整合另外两个框架;
3. 指定编译运行版本，版本锁定
    <properties>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <maven.compiler.source>1.8</maven.compiler.source>
        <maven.compiler.target>1.8</maven.compiler.target>
        <spring.version>5.1.6.RELEASE</spring.version>
    </properties>
4. 引入依赖
    1. spring-context、spring-web、spring-webmvc、spring-test(test)、junit(test)
    2. spring-jdbc、spring-aop、spring-tx、aspectjweaver
    3. javax.servlet-api(provided)、javax.servlet.jsp-api(provided)、jstl
    4. mybatis、mybatis-spring、mysql-connector-java、c3p0
5. 基本结构
    1. 实体类：cn.ssm.domain.Account
        public class Account implements Serializable {
            private int id;
            private String name;
            private double money;
            ...... //setter、getter
    }
    2. Dao层：cn.ssm.dao.IAccountDao
    3. Service层：
        cn.ssm.service.IAccountService
        cn.ssm.service.impl.AccountServiceImpl
    4. Controller层：cn.ssm.controller.AccountController

搭建 Spring 的环境

1. Spring的配置文件：resources/applicationContext.xml
    <beans xmlns="http://www.springframework.org/schema/beans"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xmlns:context="http://www.springframework.org/schema/context"
        xmlns:aop="http://www.springframework.org/schema/aop"
        xmlns:tx="http://www.springframework.org/schema/tx"
        xsi:schemaLocation="
            http://www.springframework.org/schema/beans
            https://www.springframework.org/schema/beans/spring-beans.xsd
            http://www.springframework.org/schema/context
            https://www.springframework.org/schema/context/spring-context.xsd
            http://www.springframework.org/schema/tx
            https://www.springframework.org/schema/tx/spring-tx.xsd
            http://www.springframework.org/schema/aop
            https://www.springframework.org/schema/aop/spring-aop.xsd">
        <!--Spring创建IoC容器时 要扫描哪个包下的注解-->
        <context:component-scan base-package="cn.ssm">
            <!--配置忽略的注解：Controller-->
            <context:exclude-filter type="annotation" 
                        expression="org.springframework.stereotype.Controller" />
        </context:component-scan>
    </beans>
    1. 添加 context、aop、tx 的命名空间和约束;
    2. 配置Spring创建IoC容器时，要去扫描哪些包下的注解;
    3. Controller层的注解是由SpringMVC管理的，Spring只需要关心Dao层和Service层的注解;
    4. <context:exclude-filter> 配置扫描包时忽略的内容，忽略 Controller 注解.
2. Service层
    @Service("accountService")
    public class AccountServiceImpl implements IAccountService {
        public List<Account> findAll() {
            System.out.println("findAll");
            return null;
        }
        public void save(Account account) {
            System.out.println("save");
        }
    }
3. 测试Spring环境
    ApplicationContext ac = 
            new ClassPathXmlApplicationContext("classpath:applicationContext.xml");
    IAccountService service = ac.getBean("accountService", IAccountService.class);
    service.findAll();  // findAll

搭建 SpringMVC 的环境

1. SpringMVC的配置文件：resources/springmvc.xml
    <beans xmlns="http://www.springframework.org/schema/beans"
        xmlns:mvc="http://www.springframework.org/schema/mvc"
        xmlns:context="http://www.springframework.org/schema/context"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation="
            http://www.springframework.org/schema/beans
            https://www.springframework.org/schema/beans/spring-beans.xsd
            http://www.springframework.org/schema/mvc
            https://www.springframework.org/schema/mvc/spring-mvc.xsd
            http://www.springframework.org/schema/context
            https://www.springframework.org/schema/context/spring-context.xsd">
        <!-- 开启注解扫描 -->
        <context:component-scan base-package="cn.ssm">
        <!--只去扫描Controller注解-->
            <context:include-filter type="annotation" 
                    expression="org.springframework.stereotype.Controller" />
        </context:component-scan>
        <!-- 视图解析器 -->
        <bean id="internalResourceViewResolver" 
            class="org.springframework.web.servlet.view.InternalResourceViewResolver">
            <property name="prefix" value="/pages/" />
            <property name="suffix" value=".jsp" />
        </bean>
        <!--过滤的静态资源-->
        <mvc:resources location="/css/" mapping="/css/**" />
        <mvc:resources location="/images/" mapping="/images/**" />
        <mvc:resources location="/js/" mapping="/js/**" />
        <!-- 开启SpringMVC的注解支持 -->
        <mvc:annotation-driven />
    </beans>
    1. 添加 context、mvc 的命名空间和约束;
    2. 配置扫描哪些包下的注解，SpringMVC管理Controller层，只需要关心 Controller 注解;
    3. <context:include-filter> 配置扫描包时需要管理的内容，解析 Controller 注解;
    4. 配置视图解析器，指定视图所在的目录(webapp/pages)，视图的类型(.jsp);
    5. <mvc:resources> 配置过滤静态资源，采用默认方式去处理;
    6. 开启SpringMVC的注解支持：<mvc:annotation-driven />
2. 配置 web.xml
    <web-app>
        <display-name>Archetype Created Web Application</display-name>
        <filter>
            <!--解决中文乱码的过滤器-->
            <filter-name>characterEncodingFilter</filter-name>
            <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
            <init-param>
                <param-name>encoding</param-name>
                <param-value>UTF-8</param-value>
            </init-param>
        </filter>
        <filter-mapping>
            <filter-name>characterEncodingFilter</filter-name>
            <url-pattern>/*</url-pattern>
        </filter-mapping>
        <!-- 配置控制器 -->
        <servlet>
            <servlet-name>dispatcherServlet</servlet-name>
            <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
            <!-- 让控制器去加载SpringMVC的配置文件 -->
            <init-param>
                <param-name>contextConfigLocation</param-name>
                <param-value>classpath:springmvc.xml</param-value>
            </init-param>
            <!-- 让控制器在服务器启动时创建并初始化，以便于让它去加载配置文件 -->
            <load-on-startup>1</load-on-startup>
        </servlet>
        <servlet-mapping>
            <servlet-name>dispatcherServlet</servlet-name>
            <url-pattern>/</url-pattern>
        </servlet-mapping>
    </web-app>
    1. 配置控制器DispatcherServlet，并让控制器去加载SpringMVC的配置文件;
    2. Servlet默认在第一次接收到请求时创建，配置 <load-on-startup>，让它随服务器启动就创建;
    3. 配置一个过滤器，解决中文乱码问题;
3. 在webapp目录下创建4个目录：pages、css、images、js
4. Controller层
    @Controller
    @RequestMapping("/account")
    public class AccountController {
        @RequestMapping("/findAll")
        public String findAll() {
            System.out.println("findAll");
            return "list";
        }
    }
5. 启动服务器，测试SpringMVC环境
    1. 创建 webapp/pages/list.jsp;
    2. 在 index.jsp 中添加一个 <a> 标签：<a href="account/findAll">show list</a>

搭建Mybatis环境

1. 创建Mybatis的配置文件：resources/SqlMapConfig.xml
    <!DOCTYPE configuration PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
    <configuration>
        <!-- 配置环境 -->
        <environments default="mysql">
            <!-- 配置MySQL的环境 -->
            <environment id="mysql">
                <!-- 配置事物的类型 -->
                <transactionManager type="JDBC"></transactionManager>
                <!-- 配置数据源：连接池，有三种 -->
                <dataSource type="POOLED">
                    <!-- 配置连接数据库的基本信息 -->
                    <property name="driver" value="com.mysql.cj.jdbc.Driver" />
                    <property name="url" value="jdbc:mysql:..."/>
                    <property name="username" value="root" />
                    <property name="password" value="123456" />
                </dataSource>
            </environment>
        </environments>
        <!-- 配置映射的Dao class -->
        <mappers>
            <mapper class="cn.ssm.dao.IAccountDao"/>
        </mappers>
    </configuration>
2. Dao层
    public interface IAccountDao {
        @Select("select * from account")
        List<Account> findAll();

        @Insert("insert into account(name, money) values(#{name}, #{money})")
        void save(Account account);
    }
3. 测试MyBatis环境
    InputStream is = Resources.getResourceAsStream("SqlMapConfig.xml");
    SqlSessionFactoryBuilder builder = new SqlSessionFactoryBuilder();
    SqlSessionFactory factory = builder.build(is);
    SqlSession session = factory.openSession();
    IAccountDao dao = session.getMapper(IAccountDao.class);
    List<Account> accounts = dao.findAll();
    session.close();
    is.close();

Spring 整合 SpringMVC

1. SpringMVC的配置文件springmvc.xml 中，只扫描了 Controller 注解，Spring的IoC容器中也就没有
   Service和Dao的对象，所以在控制器类中无法注入Service和Dao的对象;
2. 既然Spring的配置文件applicationContext.xml 中扫描了Service和Dao的注解，那么在启动Tomcat时，
   就去加载 Spring 的配置文件;
    1. 服务器启动时会创建一个全局的ServletContext对象，服务器关闭时，此对象才会随之销毁;
    2. 那么就可以使用一类监听器，监听ServletContext对象的创建与销毁，让监听器去加载Spring的配置文件，
    创建WEB版本的工厂，存入ServletContext域中;
    3. spring-web 提供了这样一个监听器，在 web.xml 中配置：
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>
3. 但是，ContextLoaderListener 默认加载的配置文件是：WEB-INF/applicationContext.xml
4. 除了把Spring的配置文件拷贝到 WEB-INF 目录中，还可以使用 <context-param>
    1. 在 web.xml 的<web-app>下使用 <context-param> 标签;
    2. <param-name> 配置为 contextConfigLocation 参数，<param-value>配置为Spring配置文件的位置;
    <context-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:applicationContext.xml</param-value>
    </context-param>
5. 在控制器类中使用 @Autowired 注入Service对象;
    @Controller
    @RequestMapping("/account")
    public class AccountController {
        @Autowired
        private IAccountService service;

        @RequestMapping("/findAll")
        public ModelAndView findAll() {
            List<Account> accounts = service.findAll();
            ModelAndView mv = new ModelAndView();
            mv.addObject("accounts", accounts);
            mv.setViewName("list");
            return mv;
        }
    }

Spring整合Mybatis

1. 借助 mybatis-spring 包中的对象，把Mybatis配置文件中的配置转移到Spring的配置文件中;
2. 在Spring的配置文件applicationContext.xml 中
    <!--配置连接池-->
    <bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
        <property name="driverClass" value="com.mysql.cj.jdbc.Driver" />
        <property name="jdbcUrl" value="jdbc:mysql:..." />
        <property name="user" value="root" />
        <property name="password" value="123456" />
    </bean>

    <!--配置SqlSessionFactory的工厂对象SqlSessionFactoryBean，由mybatis-spring提供-->
    <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource" />
    </bean>

    <!--配置Dao层的包，封装到mybatis-spring的MapperScannerConfigurer对象-->
    <bean id="mapperScanner" class="org.mybatis.spring.mapper.MapperScannerConfigurer">
        <property name="basePackage" value="cn.ssm.dao" />
    </bean>

    <!-- 配置声明式事务 -->
    <!--1.配置事务管理器-->
    <bean id="manager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <property name="dataSource" ref="dataSource" />
    </bean>
    <!--2.配置事务通知-->
    <tx:advice id="txAdvice" transaction-manager="manager">
        <tx:attributes>
            <tx:method name="find*" read-only="true"/>
            <tx:method name="*" isolation="DEFAULT" />
        </tx:attributes>
    </tx:advice>
    <!--3.AOP增强-->
    <aop:config>
        <!--切入点表达式-->
        <aop:advisor advice-ref="txAdvice" 
            pointcut="execution(* cn.ssm.service.impl.*.*(..))" />
    </aop:config>
3. 移除Mybatis的配置文件SqlMapConfig.xml
4. 在Service层使用 @Autowired 注入Dao对象
    @Service("accountService")
    public class AccountServiceImpl implements IAccountService {
        @Autowired
        private IAccountDao dao;

        public List<Account> findAll() {
            return dao.findAll();
        }
        public void save(Account account) {
            dao.save(account);
        }
    }
