持续集成

发表于 2016-06-02 | 分类于 JavaEE

Jenkins

1. 持续集成：Continuous Integration，简称CI，
    1. 它倡导团队中的开发人员必须经常集成他们的工作，确保软件开发的质量;
    2. 每次集成都通过自动化的构建来验证，包括自动编译、发布和测试，从而尽快地发现集成错误;
2. 持续集成的特点与作用
    1. 它是一个自动化的、周期性的集成测试过程，包括检出代码、编译构建、运行测试、结果记录、测试统计...
    2. 需要有专门的集成服务器来执行集成构建，有代码托管工具支持;
    3. 保证团队开发人员提交代码的质量，减轻软件发布时的压力;
    4. 自动化的持续集成，无需人工干预，减少重复过程，节省时间、费用和工作量.
3. Jenkins：原名Hudson，一个开源的持续集成工具;
    1. 实时监控集成中存在的错误，提供详细的日志文件和提醒功能，支持图表的形式展示项目构建的趋势和稳定性;
    2. 易安装、易配置(GUI配置界面)、变更支持、支持永久链接、集成Email/RSS/IM、Junit/TestNG测试报告;
    3. 支持分布式构建、文件指纹信息、支持第三方插件.

安装Jenkins

1. 安装JDK
    1. 在宿主机(CentOS系统)中下载rpm格式的JDK，或者上传rmp格式的JDK到宿主机上;
    2. 使用RPM的方式安装JDK，其根目录为 /usr/java/jdk1.8.0_xxx
        rpm -ivh jdk-8uxxx-linux-x64.rpm
2. 安装Jenkins
    weg https://pkg.jenkins.io/redhat/jenkins-2.180-1.1.noarch.rpm
    rmp -ivh jenkins-2.180-1.1.noarch.rpm
3. 配置用户和端口：vi /etc/sysconfig/jenkins
    JENKINS_USER="root"
    JENKINS_PORT="8888"
4. 启动服务
    systemctl start jenkins
5. 访问Jenkins Web管理平台：http://宿主机IP:8888  ==> 开始安装与配置
    1. 提示获取初始密码的路径 /var/lib/jenkins/secrets/initialAdminPassword
    2. 选择默认安装的插件：Install suggested plugins
    3. 创建管理员用户：Create First Admin User
    4. 安装完成，启动Jenkins
6. 在Jenkins Web管理平台上安装插件
    系统管理 ==> 管理插件 ==> 可选插件 ==> 搜索Maven ==> 勾选Maven Integration ==> 直接安装

全局工具配置

1. Jenkins通过Maven插件调用Maven命令、实现持续集成，而Maven插件只是Maven的入口，所以要先安装Maven;
    1. 在宿主机中下载Maven，或者上传 apache-maven-3.6.1-bin.tar.gz 到宿主机上;
        tar zxvf apache-maven-3.6.1-bin.tar.gz
        mv apache-maven-3.6.1 /usr/local/maven
    2. 配置本地仓库：vi /usr/local/maven/conf/setting.xml
        <localRepository>/usr/local/repository</localRepository>
    3. 将开发环境的本地仓库上传到宿主机上的 /usr/local/repository
2. Jenkins Web管理平台 ==> 系统管理 ==> 全局工具配置 ==> 配置JDK和Maven ==> Save
    1. 新增JDK -> 取消自动安装 -> 设置别名为jdk1.8，JAVA_HOME=/usr/java/jdk1.8.0_xxx
    2. 新增Maven -> 取消自动安装 -> 设置Name为maven，MAVEN_HOME=/usr/local/maven

Gogs与Git

1. Jenkins在持续集成时，需要从Git仓库上拉取代码，所以要为Jenkins配置Git服务器;
2. Gogs是一款极易搭建的自助Git服务，由Go语言开发，能够通过独立的二进制分发，支持Go语言支持的所有平台;
3. 在宿主机(CentOS系统)上搭建的Git没有图形界面，Gogs支持图形化操作;
4. 容器化的方式搭建Gogs
    1. 下载镜像：docker pull gogs/gogs
    2. 创建容器
    docker run -di --name=gogs -p 10022:22 -p 3000:3000 -v /var/gogsdata:/data gogs/gogs
5. 访问Gogs Web管理界面：http://宿主机IP:3000  ==> 开始安装与配置
    1. 数据库设置：SQLites3
    2. 应用基本设置 -> 域名: 宿主机的IP地址，应用URL: http://宿主机IP:3000/
    3. 管理员账号设置 -> 设置用户名、密码、邮箱 -> 立即安装;
    4. 创建远程仓库：右上角的 + 号 -> 输入仓库名称 -> 创建仓库 -> 生成仓库地址;
6. 在本地Windows系统上安装Git，为IDEA配置Git，并为工程配置远程仓库地址;
    1. File -> Settings -> Version Control -> Git -> 输入git.exe的路径;
    2. 右击工程 -> Git -> Repository -> Remotes -> 加入远程仓库地址;
7. 上传代码
    1. 右击工程 -> Git -> Add -> Commit...(提交到本地仓库) -> Reposition -> push(提交到远程仓库)
    2. 输入Gogs的管理员账号和密码 -> 上传完成;

创建任务

1. 登录Jenkins Web管理平台 -> 创建一个新任务 -> 输入名称，构建一个maven项目 -> OK
2. General：输入项目的描述内容;
3. 源码管理：选择Git，并输入远程仓库地址;
4. Build
    1. Root POM: pom.xml表示工程的pom.xml，如果想构建其子工程，如base，则输入base/pom.xml
    2. Goals and options: 输入构建时执行的Maven命令，mvn可以省略;
        clean package docker:build -DpushImage
5. 保存 -> 返回面板(回到首页) -> 生成任务列表
    1. S：灰色状态表示此任务从未执行过，蓝色状态表示上次执行是成功的;
    2. W：使用天气来表示任务执行的状态，黄色太阳表示健康，执行过程没有发生过错误;
6. 执行任务：点击列表右侧的按钮;
    1. 构建执行状态 -> 展示当前任务执行的状态 -> 点击进入控制台;
    2. 网页控制台上的输出信息，与使用IDEA构建时的控制台信息一样;

Rancher

1. Rancher：一个开源的企业级全栈化的容器部署及管理平台;
    1. Rancher为容器提供了一系列的基础架构服务：CNI兼容的网络服务、存储服务、主机管理、负载均衡、防火墙...
    2. Rancher将这些服务跨域公有云、私有云、虚拟机、物理机运行环境，真正实现一键式应用部署和管理.
2. 通俗来讲，Rancher就是Docker图形化管理工具，底层仍是Docker，它对内存的要求比较高，启动时间较慢;
3. 安装
    1. 下载Rancher镜像：docker pull rancher/server
    2. 创建容器
    docker run -di --name=rancher -p 9090:8080 rancher/server
4. Rancher Web管理平台 http://宿主机IP:9090

初始化配置

1. Default -> 环境管理 -> 添加环境
    1. 输入名称和描述，比如dev、开发环境，test、测试环境，pro、生产环境
    2. 三个环境相互隔离，互不影响.
2. 基础架构 -> 主机 - 添加主机
    1. 保存 -> 拷贝第 5 步生成的脚本，用于创建一个Rancher的代理容器;
    2. 在宿主机上执行脚本 -> 点击添加主机界面上的关闭 -> 在界面上会生成主机列表.
3. 创建应用：应用 -> 用户 -> 添加应用 -> 输入名称和描述 -> 创建;

服务部署

1. 为创建的应用添加服务：在创建的应用界面上 -> 添加服务 -> 以图形化的方式管理;
2. 部署MySQL
    1. 输入名称(mysql)和描述(mysql) -> 选择镜像: mysql-57-centos7
    2. 如果当前宿主机上已经下载过了镜像，则不要勾选 创建前总是拉取镜像，否则就要勾选;
    3. 端口映射：33306 > 3306
    4. 环境变量：MYSQL_ROOT_PASSWORD = 123456
    5. 创建 -> 在应用列表上生成 Active mysql，表示创建成功;
3. 部署RabbitMQ
    1. 选择镜像：rabbitmq:management
    2. 端口映射：5671 > 5671，5672 > 5672，4369 > 4369，15671 > 15671，15672 > 15672，25672 > 25672
    3. 创建成功后，测试访问 http://宿主机IP:15672
4. 部署Redis
    1. 选择镜像：redis
    2. 端口映射：6379 > 6379
5. 部署微服务
    1. 启动私有仓库registry、配置docker允许远程访问，为微服务添加DockerMaven插件;
    2. 刷新配置，重启容器
        systemctl daemon-reload
        systemctl restart docker
        docker start registry
        docker start rancher
    3. 在本地打包上传微服务
        mvn clean package docker:build -DpushImage
    4. 添加服务 -> 输入名称和描述 -> 选择镜像: 上传的微服务镜像 -> 映射端口: 微服务监听的端口 -> 创建;
    5. 测试访问 http://宿主机IP:映射端口

扩容与缩容

1. 并发量较高时，则需要添加更多的容器构成微服务集群；反之则缩减容器，释放内存;
2. 如果一个微服务需要扩容/缩容操作，在为应用添加微服务镜像时，不要设置映射端口，交给Rancher自动管理;
3. API -> Webhooks -> 添加接收器
    1. 类型: 扩缩容服务;
    2. 操作: 扩容(或缩容);
    3. 目标服务: 所添加的的微服务;
    4. 步长: 每次触发时新增的微服务数量;
4. 创建成功后，会展示在接收器列表上，并生成触发地址，也就是一个对外的接口;
5. 当外界应用通过 POST 请求访问 触发地址 时，Rancher就会按照设置的步长新增微服务;
6. 使用 postman 模拟：拷贝触发地址，发起POST请求，在应用列表上会反馈微服务的容器数量变化.

负载均衡器

1. 在扩容/缩容微服务时，并不知道每个微服务的端口号，也就无法访问，所以Rancher提供了负载均衡器;
2. 下拉应用界面上的 添加服务 菜单 -> 添加负载均衡 -> 输入名称和描述 -> 端口规则;
3. 端口规则：(映射)端口、目标(选择扩容/缩容的微服务)、(微服务)端口
4. 创建成功后，在应用列表上会生成负载均衡器的容器;
5. 访问测试 http://宿主机IP:映射端口

influxDB

1. influxDB：一个分布式时间序列数据库;
2. cAdvisor仅仅显示实时信息，但不存储监视数据，因此需要提供时序数据库存储cAdvisor组件的监控信息;
3. influxDB的安装
    1. 下载镜像：docker pull tutum/influxdb
    2. 创建容器
    docker run -di \
        -p 8083:8083 -p 8086:8086 \
        --expose 8090 --expose 8099 \
        --name influxsrv \
        tutum/influxdb
    3. 其中，8083端口是web访问端口，8086端口是数据写入端口;
4. 访问Web管理平台 http://宿主机IP:8083
    1. 默认数据库Database: _internal
    2. Query Templates：提供了一些方便数据库操作的菜单，可以快速输入数据库命令.

常用操作

1. 创建数据库：CREATE DATABASE "cadvisor"  --> 右上角Database下拉选择数据库
2. 查看数据库：SHOW DATABASES
3. 创建用户并授权：
    1. 创建用户:CREATE USER "username" WITH PASSWORD "password" WITH ALL PRIVILEGES
    2. 查看用户：SHOW USERS
    3. 用户授权：数据库cadvisor，用户username
        grant all privileges on cadvisor to username
        grant WRITE on cadvisor to username
        grant READ on cadvisor to username
4. 查看采集的数据：SHOW MEASUREMENTS

cAdvisor

1. cAdvisor：Google开源的用于监控基础设施应用的工具，具有强大的监控功能;
    1. cAdvisor无需任何配置就可以通过运行在Docker主机上的容器来监控docker容器;
    2. 而且，还可以监控docker主机.
2. 安装
    1. 下载镜像：docker pull google cadvisor
    2. 创建容器
    docker run --volume=/:/rootfs:ro --volume=/var/run:/var/run:rw --volume=/sys:/sys:ro
        --volume=/var/lib/docker/:/var/lib/docker:ro --publish=8080:8080 --detach=true
        --link influxsrv:influxsrv --name=cadvisor google/cadvisor -storage_driver=influxdb
        -storage_driver_db=cadvisor -storage_driver_host=influxsrv:8086
    3. --link 表示连接，连接influxDB的容器influxsrv，把采集的数据存储到其数据库cadvisor
    4. --volume ==> -v，--publish ==> -p
3. 访问Web管理平台 http://宿主机IP:8080/containers/
4. 在influxDB Web管理平台上，切换到数据库cadvisor，执行命令，查看采集的数据;
    1. 查看所有的数据：SHOW MEASUREMENTS
    2. 查看某一张表的数据：SELECT * FROM xxx

Grafana

1. Grafana：一个可视化面板(Dashboard)，有非常漂亮的图标和布局，功能齐全的度量仪表盘和图像编辑器;
    1. 支持Graphite、zabbix、influxDB、Prometheus和OpenTSDB作为数据源;
    2. 主要特征：灵活丰富的图形化选项，可以混合多种风格，支持白天和夜间模式，多个数据源.
2. cAdvisor主要负责把监控数据存储到influxDB中，Grafana则负责直观的展示influxDB中的数据;
3. 安装
    1. 下载镜像：docker pull grafana/grafana
    2. 创建容器
    docker run -d -p 3001:3000 -e INFLUXDB_HOST=influxsrv -e INFLUXDB_PORT=8086
        -e INFLUXDB_NAME=cadvisor -e INFLUXDB_USER=username -e INFLUXDB_PASS=password
        --link influxsrv:influxsrv --name grafana grafana/grafana
    3. 连接influxDB时，指定数据库名称，及其用户名和密码;
4. 访问Web管理平台 http://宿主机IP:3001/  ==> 初始用户名和密码都是admin
5. 添加数据源：Configuration -> Data Source -> Add data source
    1. Name: influxdb，type: InfluxDB  --> 添加InfluxDB作为数据源
    2. URL: http://宿主机IP:8086  --> InfluxDB的地址
    3. InfuxDB Details：输入数据库名称，及其用户名和密码;
6. 添加仪表盘：让数据以图表化的方式展示，Dashboards -> Manage
    1. +Dashboard -> 选择Graph -> Panel Title -> Edit
    2. General：输入Title
    3. Metrics
        1. Data Soruce -> 选择添加的数据源influxdb
        2. Min time inteval表示每隔一段时间采集一次数据，默认30s
        3. A：类似SQL语句
        4. A -> FROM -> default：选择查询的数据，如内存数据memory_usage
        5. A -> WHERE：筛选条件，比如按照容器名称container_name = xxx
    4. Axes：配置图表的X轴和Y轴;
    5. 点击保存图标 -> 输入New name -> Save
7. 设置预警通知：Alerting -> Alert Rules
    1. 当达到设置的阈值时，以某种方式发出通知，如邮件、Webhook
    2. Add channel -> 输入Name，Type选择webhook，Url使用Rancher的触发地址，Http Method:POST
    3. Save Test -> 发送测试 -> 查看Rancher的应用列表上，对应容器的数量是否发生了变化.
8. 使用预警通知
    Dashboards -> Manage -> 选择添加的仪表盘 -> Panel Title -> Edit -> Alert
    1. Create Alert -> Alert Config -> Conditions -> IS ABOVE，设置阈值;
    2. Notifications -> Send to，添加设置的预警通知;
    3. Message的内容只有在使用邮件通知时才需要设置;
    4. 重新保存.
