SpringBoot扩展

SpringBoot

  1. Spring4.3中引进了@GetMapping、@PostMapping、@PutMapping、@DeleteMapping、@PatchMapping 来帮助简化常用的HTTP方法的映射;
    1. @GetMapping是一个组合注解，等效于 @RequestMapping(method=RequestMethod.GET);
    2. @PostMapping是 @RequestMapping(method=RequestMethod.POST) 的缩写;
  2. @RestController：相当于 @ResponseBody + @Controller 的组合;
    @RestController
    @RequestMapping(“/user”)
    public class RestController {
    @GetMapping("/getList")
public List<User> getList() {
    List<User> us = dao.findAll();
    return us;
}
@GetMapping("/test")
public List<User> test() {
    return "Hello Boot";
}
    }
    1. @RestController 的方法都只能返回String、Object、Json等实体对象;
    2. 方法返回的对象数据会被jackson转为JSON字符串，再写入响应体中;
    3. 如果想要跳转页面，则用 ModelAndView 封装;
      @GetMapping(“/index”)
      public ModelAndView toIndex() {
      ModelAndView mv = new ModelAndView("index");
return mv;
      }
  3. SpringBoot中，默认采用Servlet3.0，它可以完全使用注解取代web.xml了，所以SpringBoot没有了web.xml;

    Spring的属性的注解注入的示例

  4. Spring
    1. Spring1.0时代因为JDK1.5刚出来，注解开发尚未盛行，所以一切Spring配置皆是XML格式;
    2. Spring2.0时代引入了注解开发，但并未完善，因此并未完全替代XML;
    3. Spring3.0时代的注解已经非常完善了，因此Spring推荐使用注解(Java配置)代替XML;
  5. Druid是Java语言中最好的数据库连接池，能够提供强大的监控和扩展功能;

  6. Java配置主要依靠Java类和注解，比较常用的注解
    1. @Configuration：声明一个类为配置类，代替XML配置文件；
    2. @Bean：声明在方法上，将方法的返回值加入到IoC Bean容器，代替标签
    3. @Value：属性注入;
    4. @PropertySource：指定外部属性配置文件;

  7. 创建一个JDBC相关的配置类

    1. 创建resources/jdbc.properties
      jdbc.driverClassName=com.mysql.cj.jdbc.Driver
      jdbc.url=jdbc:mysql://host:port/database?serverTimezone=Asia/Shanghai
      jdbc.username=root
      jdbc.password=123456
      @Configuration
      @PropertySource(“classpath:jdbc.properties”)
      public class JdbcConfig {
      @Value(“${jdbc.driverClassName}”)
      private String driverClassName;
      @Value(“${jdbc.url}”)
      private String url;
      @Value(“${jdbc.username}”)
      private String username;
      @Value(“${jdbc.password}”)
      private String password;

      @Bean
      public DataSource getDataSource() {

      DruidDataSource ds = new DruidDataSource();
ds.setDriverClassName(driverClassName);
ds.setUrl(url);
ds.setUsername(username);
ds.setPassword(password);
return ds;

      }
      }

  10. 创建JdbcProperties，使用 @ConfigurationProperties 注解，把 application.properties 中的键值映射到Java类的属性上

    1. 引入lombok依赖
      org.projectlombok
      lombok
      1.18.6
      provided
    2. lombok能够通过注解的方式，在编译时自动为属性生成构造器、getter/setter、equals、hashcode、toString()
    3. 但是在源码中并没有getter和setter方法，但是在编译生成的字节码文件中有getter和setter方法，使代码更加简洁
    4. 使用 @Data 注解
      @ConfigurationProperties(prefix=”jdbc”) –>声明属性的前缀为 jdbc
      @Data
      public class JdbcProperties {
      private String driverClassName;
      private String url;
      private String username;
      private String password;
      }
    5. 还可以在 JdbcProperties 上使用 @Component 注解，把JdbcProperties添加进Spring IoC容器中;

    6. 除此之外，还可在 JdbcConfig 中，通过 @EnableConfigurationProperties 引入 JdbcProperties，
      这样JdbcConfig中注入 JdbcProperties 对象;
      @Configuration
      @EnableConfigurationProperties(JdbcProperties.class)
      public class JdbcConfig {
      @Autowired
      private JdbcProperties jdbc; //1. @Autowired的注入方式

      public JdbcConfig(JdbcProperties jdbc) { //2.构造方法的注入方式

      }

      @Bean
      public DataSource getDataSource(JdbcProperties prop) { //3.方法参数的注入方式

      DruidDataSource ds = new DruidDataSource();
ds.setDriverClassName(prop.getDriverClassName());
ds.setUrl(prop.getUrl());
ds.setUsername(prop.getUsername());
ds.setPassword(prop.getPassword());
return ds;

      }
      }

  11. 如果只有一个Bean对象需要使用这些配置，还可以更优雅
    1. application.properties 的配置不变
    2. 不需要属性映射类JdbcProperties，直接在对应Bean对象上使用 @ConfigurationProperties，实现属性映射
      @Configuration
      public class JdbcConfig {
      @Bean
      @ConfigurationProperties(prefix=”jdbc”)
      public DataSource getDataSource(JdbcProperties prop) {
      return new DruidDataSource();  //配置文件中的key名与DruidDataSource中的属性名保持一致，就可以实现键值的映射
      }
      }

      整合Mybatis

  17. SpringBoot没有为Mybatis提供启动器，但Mybatis自己写了SpringBoot的启动器：mybatis-spring-boot-starter
  18. 凡是Spring提供的启动器，都是以spring开头的，框架自己写的启动器都不是以spring开头的
  19. 开启自动驼峰匹配
    1. 原有的配置文件中的配置方式



    2. 在application.yaml中配置
      mybatis:
      configuration:
      map-underscore-to-camel-case: true
      mapper-locations: mapper/.xml //指定映射配置文件的位置：resources/mapper/.xml
      type-aliases-package: cn.test.domain //实体类的别名扫描包名，为POJO配置别名
  20. 在Mybatis中，Dao层接口也称为Mapper，配置Mapper有两种方式
    1. 在每个Mapper上使用 @Mapper 注解
      @Mapper
      public interface UserMapper {}
    2. 在项目的启动器类上使用 @MapperScan 注解，指定Mapper所在的包
      @SpringBootApplication
      @MapperScan(“cn.test.mapper”)
      public class QuickBootApplication {
      …
      }

  21. 通用Mapper：一款中国人自己开发的框架，只支持单表的CRUD

    1. 通常单表的SQL语句的性能是固定的，不像多表的SQL语句那样，有很大的优化余地；
    2. 所以在单表操作时，完全可以自动生成SQL语句，无需再去为Dao创建对应的映射配置文件；
    3. 这就是通用Mapper的作用
    4. 那么也就无需在application.yaml中配置mybatis.mapper-locations了；
    5. 通用Mapper的启动器：mapper-spring-boot-starter
    6. Dao层接口需要继承Mapper，泛型为实体类
      public interface UserMapper extends Mapper {}
    7. 其底层利用Mybatis的拦截器，动态生成SQL语句，并做了SQL缓存，所以性能很好；
    8. 通用Mapper的启动器中已经引入了spring-boot-starter、spring-boot-starter-jdbc、mybatis、mybatis-spring
    9. 所以，不能再引入mybatis-spring-boot-starter、spring-boot-starter-jdbc
    10. 而且通用Mapper还会开启驼峰匹配，无需在application.yaml中自己开启了；
    11. 同时，@MapperScan、@Mapper 要使用通用mapper包下的注解，不能再使用Mybatis包下的了
    12. 实体类相关
      1. 通用Mapper默认会使用实体类名作为数据库表名，@Table(name=”表名”)用于指定数据库表名；
      2. @Id 注解在属性上，指定主键；
      3. 对于自增长的主键，需要使用 @KeySql(userGeneratedKeys=true) 注解
      4. 如果实体类的某个属性不作为数据库的字段，则使用 @Transient 注解
        @Data
        @Table(name=”tl_user”)
        public class User {
        @Id
        @KeySql(userGeneratedKeys=true)
        private Integer id;
        private String username;
        @Transient
        private String remark;
        }

跨域问题

  1. 浏览器的同源策略所致，同源是指域名/IP、协议、端口都相同，不同源的客户端脚本在没有被明确授权的情况下，不能读写对方的资源;
  2. 主域名和子域名之间也是跨域，cookie也无法访问;
  3. 同源策略限制的行为：
    1. Cookie、LocalStorage 和 IndexDB 无法读取;
    2. DOM 和 JS对象无法获得;
    3. AJAX 请求不能发送.
  4. 注意：
    1. 浏览器并没有限制跨站请求的发起，而是返回结果被浏览器拦截了，比如CRSF跨站攻击原理，无论是否跨域，请求已经发给了服务器;
    2. 但是，有些浏览器不允许从HTTPS跨域访问HTTP，如Chrome和Firefox，它们会拦截请求的发出;
  5. 解决跨域问题的方案有很多：
    1. 通过jsonp跨域
    2. document.domain + iframe跨域
    3. location.hash + iframe
    4. window.name + iframe跨域
    5. postMessage跨域
    6. 跨域资源共享(CORS)
    7. 前端通过Nginx解决跨域问题
    8. nodejs中间件代理跨域
    9. WebSocket协议跨域
  6. SpringMVC解决跨域问题：JSONP、CORS、WebSocket
  7. WebSocket是一种通信协议，使用ws://(非加密)和wss://(加密)作为协议前缀;
    1. 在2008年诞生，2011年成为国际标准，所有浏览器都已经支持了;
    2. WebSocket属于服务器推送技术的一种，最大特点就是，保持长链接，服务器和客户端都可以主动向对方推送/发送信息，是真正的双向平等对话;
    3. WebSocket协议不实行同源政策，只要服务器支持，就可以进行跨源通信;

      CORS

  8. CORS是一个W3C标准，称为跨域资源共享;
  9. CORS需要浏览器和服务器同时支持;
    1. 所有浏览器都支持该功能，IE浏览器不能低于IE10;
    2. 整个CORS通信过程都是浏览器自动完成，不需要用户参与;
    3. 对于开发者来说，CORS通信与同源的AJAX通信没有差别，代码完全一样;
    4. 浏览器一旦发现AJAX请求跨源，就会自动添加一些附加的头信息，有时还会多出一次附加的请求，但用户不会有感觉;
    5. 实现CORS通信的关键是服务器，只要服务器实现了CORS接口，就可以跨源通信;
  10. CORS与JSONP相比：
    1. JSONP只能实现GET请求，而CORS支持所有类型的HTTP请求;
    2. 使用CORS，开发者仍可以使用普通的AJAX发起请求和获得数据，比起JSONP有更好的错误处理;
    3. JSONP主要被老浏览器支持，它们往往不支持CORS，而绝大多数现代浏览器都已经支持了CORS;
  11. SpringMVC4.2开始，支持使用 @CrossOrigin 解决跨域问题；
    1. @CrossOrigin 可以注解在控制器类/方法上，默认允许在@RequestMapping中指定的所有源和HTTP方法;
    2. value、origins属性：允许可访问的域列表;
    3. maxAge属性：准备响应前的缓存持续的最大时间(s);
    4. allowedHeaders、exposedHeaders、methods、allowCredentials
    5. 在控制器方法上使用时，可能出现一些附带问题
      1. @RequestMapping中需要声明method属性，否则@CrossOrigin失效;
      2. session失效：AJAX中需要添加 xhrFields:{withCredentials:true}，以保证Session的一致性;
      3. withCredentials:true 表示对附带身份凭证的请求，服务器不得设置 Access-Control-Allow-Origin=””,
        因为请求头中携带了Cookie，如果Access-Control-Allow-Origin=””，请求将会失败;
        而设置 Access-Control-Allow-Origin=”http://foo.example"，则会请求成功；
        其次服务器需要设置 Access-Control-Allow-Credentials:true，否则响应内容不会返回给客户端;
    6. 如果控制器类和方法上都是用了@CrossOrigin，Spring会合并这两个注解中设置的属性;
  12. SpringMVC的XML配置文件中，全局配置CORS
    mvc:cors
    <mvc:mapping path="/**" />
    </mvc:cors>
    1. 更加细粒度的配置：
      mvc:cors
      <mvc:mapping path=”/api/**”
      allowed-origins="http://domain1.com, http://domain2.com"
allowed-methods="GET, PUT"
allowed-headers="header1, header2, header3"
exposed-headers="header4, header5" allow-credentials="false"
max-age="3600" />
      <mvc:mapping path=”/resources/**”
      allowed-origins="http://domain1.com" />
      </mvc:cors>
  13. 如果不支持@CrossOrigin，可以在Filter中拦截此请求，添加请求头：setHeader(“Access-Control-Allow-Origin”, “*”);
